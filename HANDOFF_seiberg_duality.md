## Goal
Let the toric (p,q)-web tab **cycle through the distinct toric (Seiberg-dual) phases** of a
diagram, each shown as a full brane tiling, generated by **urban renewal** on the dimer.
Canonical check: F0 (square `(-1,0),(1,0),(0,1),(0,-1)`) has phase I (4 nodes / 8 fields /
square faces) and phase II (4 nodes / 12 fields / hexagonal). A geometric *flop* correctly
does NOT change the dimer (Kähler move); Seiberg duality is the operation that does.

## VALIDATED but NOT shipped: the Seiberg/meson move (quiver level is correct)
`_seiberg_meson_prototype.py` (dev script, run it) implements urban renewal on a square face
via the gauge-theory meson rule and **produces the correct F0 phase II quiver**:
- A square gauge face = a φ-orbit of 4 darts → edges with corners `b1:(e0,e3) w0:(e3,e7)
  b0:(e7,e4) w1:(e4,e0)`; node0 has incoming X={e3,e4}, outgoing Y={e0,e7}.
- Move: reverse a's 4 arrows (x3,x4,y0,y7), add the 2×2 mesons (m37,m40,m74,m03), rebuild the
  superpotential terms (old corner pair `YX`→meson; new cubic `M·y·x` terms). For F0 this is
  +4 fields, +4 vertices, no mass integration.
- Result: a **genus-1 torus dimer, 4 gauge nodes, 12 fields, anomaly-free, valid two-term
  (toric) superpotential**, adjacency = F0 phase II (the node with 4 incoming mesons). The
  prototype finds **54** valid (genus-1 + anomaly-free + toric) cyclic-order assignments.

## THE OPEN PROBLEM (the only blocker): edge homology of the dual tiling
Of those 54 valid phase-II embeddings, **0 Newton-certify** back to the F0 square. The new
(meson/reversed) edges' **homology cochain** is wrong: the naive rule
`h(meson)=h(Y)+h(X)`, `h(reversed)=−h` does not satisfy the torus face-closure, so
`kasteleyn_newton_polygon(dual)` ≠ F0 diagram even though the quiver is right.

Homology is gauge-dependent (coboundary shifts don't change the Newton polygon up to
translation), but the two global **periods** must match the true torus windings. The fix is to
**solve for homology from the rotation system**: enforce that every face (φ-orbit) closes
(signed edge-vector sum = 0), gauge-fix a spanning tree to 0, and fix the 2 residual periods so
the Kasteleyn Newton polygon equals the input diagram (it is GL(2,Z)-invariant, so any
primitive period basis consistent with face-closure should land on a GL-equivalent polygon).
This is the analogue of the spanning-tree lift in `_try_tiling` (inverse.py ~L267-349) but
driven by the combinatorial map instead of the zig-zag arrangement.

Alternative: obtain the exact urban-renewal edge/homology recipe from **Franco et al.
hep-th/0504110 "Brane Dimers and Quiver Gauge Theories", Fig. 2** (the web sources all defer to
it; the arXiv PDF would not decode through the fetch tool this session).
